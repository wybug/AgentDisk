### 前置条件

| 服务 | 端口 | 启动命令 |
|------|------|----------|
| MySQL 8.0 | 3306 | 已部署 |
| MinIO | 9000 | 已部署 |

### 配置确认

1. `config.yaml` 中 `oauth2.enabled` 设为 `true`
2. `oauth2.auth_url` 指向 `http://localhost:3000/oauth2/authorize`
3. `oauth2.redirect_url` 指向 `http://localhost:5173/auth/callback`
4. 环境变量 `DB_PASSWORD`、`OSS_ACCESS_KEY`、`OSS_SECRET_KEY` 已设置

## 测试依赖关系

测试按 T01→T19 顺序执行，存在数据依赖，不可乱序或单独运行中间用例。

```
T01 (Admin Bootstrap，独立)
├── T02a (Admin 登录 + 侧边栏导航，依赖 T01)
├── T02b (Admin 公共目录管理，依赖 T01)
├── T02c (Admin API Key 管理，依赖 T01)
├── T02d (用户端公共目录浏览，依赖 T02b)
├── T02e (Admin 清理 + 退出登录，依赖 T02b/T02c)
├── T02f (Admin MFA 通行密钥管理，依赖 T01，需人工配合 WebAuthn 原生弹窗)
└── T02g (Admin MFA 登录验证，依赖 T01，需人工配合 WebAuthn 原生弹窗)
T03 (登录 + 全局清理)
├── T04 (文件夹，自包含)
├── T05 (文件上传，创建 3 个测试文件)
│   ├── T06 (文件预览，依赖 T05 的文件)
│   ├── T07 (文件下载，依赖 T05 的文件)
│   ├── T09 (标签管理，依赖 T05 的文件)
│   ├── T11 (分享管理，依赖 T05 的文件)
│   └── T12 (权限管理 + 清理 T05 文件，依赖 T05 的文件，末尾删除全部文件和回收站)
├── T08 (回收站，自建测试文件)
├── T10 (版本回滚，自建测试文件)
├── T13 (存储空间，自建测试文件)
├── T14 (网关管理，独立)
├── T15 (跨用户隔离，独立)
└── T16 (页面导航，独立)
T17 (最终验证，检查数据污染)
T18 (网关 Agent 管理，独立)
T19 (Chat ReturnFile + Markdown 渲染，独立)
```

**清理规则**：
- T03 负责全局清理（删除所有文件/文件夹 + API 清理分享/权限/回收站），为后续用例提供干净初始状态
- T05 上传的文件由 T12 在权限测试完毕后统一删除并清理回收站
- 各用例验证完毕后自行清理创建的数据，避免污染后续用例
- T02f/T02g 自行清理（关闭 MFA、删除所有通行密钥），确保不影响其他 Admin 用例
- T17 不做清理，仅验证所有数据已归零（回收站、分享、权限、文件、文件夹）
