# 版本回溯

AgentDisk 为文件提供自动版本管理功能。每次更新文件时，系统会自动创建版本快照，你可以查看完整的版本历史并随时回滚到任意历史版本。

## 自动版本快照

当你通过以下方式更新文件时，系统会自动创建版本快照：

- **上传同名文件**：向同一目录上传与已有文件同名的文件
- **API 更新文件**：调用 `PUT /v1/disk/files/:id` 接口更新文件内容

版本快照创建流程：

1. 系统保存当前文件的 OSS 对象作为历史版本
2. 记录版本元数据（版本号、文件大小、MD5、操作者）
3. 新文件内容覆盖当前版本
4. 文件的 `version` 字段自动递增

::: tip
版本快照是自动的，无需手动触发。每次文件内容变更都会生成新版本。
:::

## 查看版本历史

### 通过 Web 界面

1. 在文件列表中点击目标文件
2. 在文件详情中查看 **版本历史** 标签页
3. 可以看到该文件的所有历史版本，包括版本号、文件大小、创建时间
4. 选择任意版本可以查看详情或执行回滚

### 通过 API

```bash
# 查看指定文件的版本历史
curl "http://localhost:9100/v1/disk/versions?fileId=5" \
  -H "Authorization: Bearer $TOKEN"
```

返回示例：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "fileId": 5,
      "userId": "user001",
      "version": 1,
      "ossKey": "user001/files/abc123_v1.pdf",
      "fileSize": 1048576,
      "md5": "d41d8cd98f00b204e9800998ecf8427e",
      "snapshotBy": "user001",
      "createdAt": "2026-01-10T08:00:00Z"
    },
    {
      "id": 2,
      "fileId": 5,
      "userId": "user001",
      "version": 2,
      "ossKey": "user001/files/abc123_v2.pdf",
      "fileSize": 2097152,
      "md5": "7d793037a0760186574b0282f2f435e7",
      "snapshotBy": "user001",
      "createdAt": "2026-01-12T14:30:00Z"
    }
  ]
}
```

## 版本元数据

每个版本快照包含以下信息：

| 字段 | 说明 |
|------|------|
| `id` | 版本快照 ID |
| `fileId` | 关联的文件 ID |
| `userId` | 创建该版本的用户 ID |
| `version` | 版本号（从 1 开始递增） |
| `ossKey` | 该版本在 OSS 中的存储路径 |
| `fileSize` | 文件大小（字节） |
| `md5` | 文件 MD5 校验值 |
| `snapshotBy` | 触发快照的操作者 |
| `createdAt` | 版本创建时间 |

通过 MD5 字段可以判断两个版本的内容是否相同，通过 `fileSize` 可以快速了解文件大小的变化趋势。

## 回滚到历史版本

### 通过 Web 界面

1. 打开文件的版本历史
2. 找到要回滚的目标版本
3. 点击 **回滚** 按钮
4. 确认回滚操作

回滚后：
- 当前文件内容替换为目标版本的内容
- 系统会为回滚操作创建一个新的版本快照
- 原版本历史记录保持不变

### 通过 API

```bash
curl -X POST http://localhost:9100/v1/disk/versions/rollback \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "fileId": 5,
    "versionId": 1
  }'
```

返回示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 5,
    "fileName": "报告.pdf",
    "version": 3,
    "fileSize": 1048576,
    "md5": "d41d8cd98f00b204e9800998ecf8427e",
    "updatedAt": "2026-01-15T16:00:00Z"
  }
}
```

::: warning
回滚操作会将当前版本的内容替换为历史版本的内容。虽然回滚前的版本仍保留在版本历史中，但建议在回滚前确认操作。
:::

## 版本管理流程示例

以下是一个典型的文件版本生命周期：

```
时间线：
├── 2026-01-10  上传初始文件 → 版本 1
├── 2026-01-12  更新文件内容 → 版本 2（版本 1 自动快照）
├── 2026-01-14  再次更新文件 → 版本 3（版本 2 自动快照）
├── 2026-01-15  发现版本 2 有误，回滚到版本 1 → 版本 4（内容与版本 1 相同）
└── 可以随时查看版本 1~4 的完整历史
```

## 存储说明

- 每个版本在 OSS 中独立存储，不会相互覆盖
- 版本快照不影响用户空间配额的计算
- 建议定期清理不需要的历史版本，释放 OSS 存储空间
