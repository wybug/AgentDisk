# AgentDisk 智能体团队约束文件

## 1 概述

本文档为 **AgentDisk 智能体云盘** 项目强制开发规范，用于多智能体 Team 协同研发。因项目模块多、上下文负载大、安全要求高，禁止单智能体包揽开发，强制模块拆分、职责隔离、独立验收。本规则对所有研发智能体永久生效，所有 DoD 不通过不得合并、不得上线。

## 2 团队拆分架构（固定不变）

整体采用 **1主控 + 6垂直业务智能体**，单向依赖、同级隔离、禁止跨模块篡改。

|编号|智能体名称|职责域|依赖|
|---|---|---|---|
|T0|调度主控智能体|任务分发、规范管控、合并验收、风险汇总|无|
|T1|底层底座智能体|数据库、OSS封装、全局基础组件|T0|
|T2|核心文件接口智能体|目录、文件基础CRUD、OpenAPI|T1|
|T3|权限安全智能体|RBAC、鉴权、拦截、安全合规|T2|
|T4|高级能力智能体|版本回溯、回收站、标签检索|T2|
|T5|分享预览智能体|在线预览、外链分享、时效管控|T4|
|T6|协议联调智能体|协议改造、兼容、全链路测试、灰度|T3、T5|

## 3 本地开发环境准备

### 3.1 Redis

- 安装：`brew install redis`
- 启动：`redis-server`
- **所有测试运行前必须使用 `redis-server` 启动**，禁止使用 `brew services start redis`，禁止跳过或 mock。

### 3.2 其他依赖

- MySQL 8.0 或 SQLite
- MinIO 或兼容 OSS 服务

## 4 通用强制开发规范（所有智能体）

### 4.1 Worktree 分支隔离规范

- 采用 Git Worktree 多工作区开发，每个智能体独占独立工作目录+独立分支。
- main 主干禁止直接提交，必须经过合并请求 & DoD 验收。
- 仅 T0 拥有代码合并权限，子智能体不可私自合并主干。
- 禁止跨目录修改他人模块代码、配置、资源。

### 4.2 单元测试强制规范

- 所有模块必须编写单元测试，业务代码覆盖率 >= 80%。
- 工具类、枚举、公共方法覆盖率必须 100%。
- 禁止注释、跳过、屏蔽测试用例。
- 所有测试必须本地、CI 环境全部 Pass。
- **`go test ./...` 必须全部通过，任何用例失败禁止合并、禁止交付。**
- 运行测试前必须先启动 `redis-server`，否则测试直接失败。

### 4.3 静态检查强制规范

- 所有模块代码必须通过 `golangci-lint run ./...` 检查，零 Error、零 Warning 方可合并。
- 使用项目根目录 `.golangci.yml` 统一配置，禁止私自修改 lint 规则。
- 安装命令：`make lint-install`，运行命令：`make lint`。
- 新增代码提交前必须本地运行 `make lint` 并确认通过。
- lint 未通过等同于测试未通过，禁止合并、禁止交付。

### 4.4 接口设计规范

- **统一响应格式**：所有接口必须使用 `pkg/response` 中的 `OK`、`Created`、`Fail`、`BadRequest`、`Unauthorized`、`Forbidden`、`NotFound`、`InternalError` 返回，禁止直接使用 `c.JSON()` 构造响应体。标准响应结构为 `{"code":int,"message":string,"data":any}`。
- **命名风格一致**：请求/响应 JSON 字段使用 camelCase（如 `folderId`、`fileName`、`downloadToken`），query 参数使用完整可读名称（如 `folderId`、`parentId`、`tags`、`token`），禁止使用缩写（如 `t`、`fid`、`uid`）。
- **错误响应分级**：客户端错误用对应语义方法（400/401/403/404），服务端内部错误统一使用 `InternalError`，禁止将内部错误详情、堆栈信息暴露给客户端。
- **资源路由规范**：RESTful 风格，资源名用复数名词（`/folders`、`/files`、`/shares`），动作用子路径（`/tags/bind`、`/tags/search`、`/files/upload`）。
- **鉴权一致性**：需鉴权接口统一由路由中间件 `HybridAuth` 处理，公开接口（如下载、外链访问）需在 router 中明确注册为公开路由，禁止在 handler 内部自行判断鉴权跳过。
- **新增接口必须同步编写单元测试和集成测试**，覆盖正常流程和错误分支。

### 4.5 部署权限隔离规范

- 每个智能体仅允许编译、部署、修改自身负责模块。
- 公共依赖仅 T1 可修改，其他智能体只读。
- 禁止私自修改全局配置、密钥、OSS策略、鉴权逻辑。

### 4.6 通用安全规范

- 全部接口强制 Bearer Token 鉴权。
- 所有数据强制携带 userId 隔离，禁止通用无过滤SQL。
- OSS 永久私有，禁止裸链、禁止公开读写。
- 敏感数据脱敏、异常堆栈禁止对外暴露。
- 高危操作必须留存审计日志，保留90天以上。

### 4.7 信息安全规约

- **严禁读取 `.env` 文件内容**，`.env` 包含数据库密码、OSS密钥、JWT签名等核心凭证，任何智能体不得通过 Read、cat、grep 等任何方式读取或输出该文件。
- **严禁在日志、响应、调试信息中泄露密钥、Token、密码**等敏感字段。
- **严禁读取系统敏感文件**，包括但不限于 `/etc/shadow`、`~/.ssh/`、`~/.aws/`、各类 credentials 文件。
- **严禁执行 `env`、`printenv`、`export` 等命令**输出环境变量。
- 所有凭证仅通过环境变量注入，代码中禁止硬编码，日志中禁止明文打印。

## 5 各智能体 DoD 完工标准（硬性卡点）

### 5.1 T0 主控智能体 DoD

- 完成任务拆解、依赖梳理、风险清单。
- 统一全部编码规范、文档规范、命名规范。
- 逐条核验所有子智能体 DoD，不合格禁止流转。
- 最终合并完整工程包、归档全部文档。

### 5.2 T1 底层底座 DoD

- 8张数据表完整设计，索引、备注、默认值齐全。
- OSS工具类：上传、删除、签名、拷贝、权限控制完成。
- 全局返回体、异常拦截、常量枚举完成。
- 数据库、OSS、配置本地连通验证通过。

### 5.3 T2 核心文件接口 DoD

- 全部 OpenAPI 接口开发完成，严格对齐 OpenAI 风格。
- 目录、文件完整CRUD，OSS路径规则强制落地。
- 智能体产物自动入盘、闲聊过滤逻辑生效。
- 入参校验、防重、容错全部完成。

### 5.4 T3 权限安全 DoD

- RBAC权限模型完成，四种权限：owner/read/write/delete。
- 默认权限规则固化，跨用户强制隔离。
- 越权测试、篡改ID测试全部拦截。
- OSS签名时效、私有管控、安全脱敏全部完成。

### 5.5 T4 高级能力 DoD

- 版本回溯：自动快照、版本列表、一键回滚。
- 回收站：逻辑删除、恢复、彻底销毁、日志留存。
- 标签检索：绑定、解绑、多条件筛选、模糊查询。
- 杜绝慢查询，大数据量检索性能合格。

### 5.6 T5 分享预览 DoD

- 在线预览：MD/代码/文本/图片沙箱渲染。
- 外链分享：有效期、提取码、访问次数限制。
- 外链手动作废、自动过期、防爬防盗。
- 访问日志完整溯源。

### 5.7 T6 协议联调 DoD

- 协议新增 userId、cloudDisk 字段，向下兼容旧会话。
- 多智能体共享文件切换验证通过。
- 全链路流程：生成-保存-读取-分享-预览跑通。
- 灰度开关、降级、熔断、压力测试完成。

## 6 红线禁止事项（永久不可违反）

1. 禁止单智能体包揽多模块开发。
2. 禁止修改非自身职责代码、配置、数据表。
3. 禁止硬编码密钥、路径、敏感配置。
4. 禁止OSS公开权限、禁止裸链暴露资源。
5. 禁止跳过鉴权、绕过权限拦截。
6. 禁止屏蔽单元测试、强行交付。
7. 禁止破坏协议向下兼容机制。
8. 禁止在测试未全部通过时提交代码或发起合并请求。
9. 禁止在 lint 检查未通过时提交代码或发起合并请求。
